# OpenSCAD / BOSL2 Technical Protocol v1.0
## Инженерный регламент для генерации Cookie Cutter (MDS v3.0)

### Раздел 1: Технический стек и абсолютные запреты (пп. 1-15)
1. Библиотека: `include <BOSL2/std.scad>` (кроме отладочного режима `К9_DEBUG`).
2. Стандартные примитивы ЗАПРЕЩЕНЫ: `cube()`, `cylinder()`, `sphere()`.
3. Булевы операции ЗАПРЕЩЕНЫ: `union()`, `difference()`, `intersection()`.
4. Разрешены: `cuboid()`, `cyl()`, `prismoid()`, `diff()`, `tag("remove")`, `tag("keep")`.
5. ЗАПРЕЩЕНЫ: `up()`, `down()`, `move(z=...)`. Только `move([0,0,value])`.
6. ЗАПРЕЩЕНЫ: `minkowski()`, `hull()` для создания скруглений.
7. Скругления плоских деталей: `offset(r=R) square()` + `linear_extrude`.
8. Всегда используй `$slop = 0.01`; в начале файла для микро-зазоров.
9. Радиусы всегда защищены: `max(r, 0.01)`. Запрещен `circle(r=0)` или `cylinder(d=0)`.
10. ЗАПРЕЩЕНЫ: `polyhedron()`, `surface()`. Геометрия через `polygon()` и `linear_extrude()`.
11. ЗАПРЕЩЕНО: `rotate_extrude()`. Для дуг использовать `polygon()`.
12. Для всех вычисляемых радиусов скруглений обязателен `max(0, ...)`.
13. Флаг `$fn` должен быть параметризован в пресетах.
14. Для дуг (сектор кольца) `$fn` принудительно задается равным 360.
15. Запрещено смешивать `method="smooth"` и `radius=` в `round_corners()`.

### Раздел 2: Cookie Cutter Engine — Конструкция (пп. 16-35)
16. Тип: Cookie Cutter (штамп-вырубка). Печать: ножом вверх (основанием на стол).
17. Материал: PLA. PETG ЗАПРЕЩЕН (тянет глину).
18. Нож (Blade): всегда прямой вертикальный. Толщина `blade_w = 0.9`.
19. ЗАПРЕЩЕНЫ любые фаски на ноже в геометрии. `cutter_chamfer` всегда равен 0.
20. Фланец (Flange): плоское основание, отогнутое НАРУЖУ от стенки на 4 мм. Толщина 2 мм.
21. Фланец = `offset(delta=blade_w/2+flange_w)` минус `offset(delta=-blade_w/2)`.
22. Центральное ребро жесткости (Rib): ширина `rib_w = 8`, через `intersection()` с центральной полосой.
23. Ребра жесткости внутри дуги (`ribs_solid_raw` / `ribs_split_raw`) — поперечные балки.
24. Внутренние ребра: ширина `rib_w` (8 мм). Крайние ребра: `edge_rib_w` (3 мм).
25. В разрезном режиме крайние ребра толще: `edge_rib_w_split` (6 мм).
26. Все компоненты штампа (Blade, Flange, Ribs) объединяются в один `union()`.
27. Геометрия строится в локальных координатах, затем смещается к (0,0) одним `translate()`.
28. Шов внахлест не используется (`seam_overlap = 0`).
29. Скос шва 45° не закладывается в геометрию (делается струной вручную).
30. Ориентационные метки: текст `>` на фланце через равные промежутки.
31. Для меток использовать шрифт `Golos Text:style=Bold`, размер 5.
32. Клеймо (Label): `linear_extrude(0.6)` текста, ограниченное `intersection()` с фланцем.
33. Отверстие для подвешивания: опционально, по умолчанию закомментировано.
34. Пины (Pins): горизонтальные, на широкой грани (`rotate([90, 0, 0])`).
35. Пины для донных каттеров генерируются отдельным файлом.

### Раздел 3: Геометрия дуги и развертки (пп. 36-50)
36. Развертка стенки: сектор кольца (усеченный конус).
37. Сложные профили (bell, camp, latte) ЗАПРЕЩЕНЫ. Только усеченный конус.
38. Если `D_top == D_bottom` — развертка это прямоугольник (`square()`).
39. Радиусы дуги: `R_outer` и `R_inner` вычисляются из длин окружностей и высоты стенки.
40. Угол сектора: `angle = (Outer_Circ / R_outer) * (180 / PI)`.
41. Дуга строится через `polygon(points)` без `intersection()` и `difference()`.
42. Точки дуги генерируются через `for` с шагом, обеспечивающим гладкость ($fn=360).
43. Смещение в (0,0): вычислить min/max полигона, сдвинуть на `- (min + max) / 2`.
44. Хорда дуги (`Chord = 2 * R * sin(angle/2)`) используется для проверки размещения на столе.
45. Длина дуги (`Arc_Length = 2 * PI * R * angle / 360`) используется для расчета количества ребер.
46. Отверстия и вычитания: высота с запасом `panel_thick + 0.2`, старт с `-0.1`.
47. При вычитании цилиндров никогда не полагаться на точное совпадение граней.
48. Для центрирования сетки отверстий использовать формулу: `(panel_size - (count-1)*spacing)/2`.
49. Обязателен `assert(offset >= 0)` для всех вычисленных отступов.
50. Фаска добавляется только после проверки базовых сквозных отверстий.

### Раздел 4: Система разрезания и склейки (пп. 51-65)
51. Резак разрезается, если хорда дуги > эффективного размера стола (196 мм).
52. Количество частей: `split_parts = 3`. Зазор между частями: `part_gap = 40`.
53. Разрезание: `intersection()` полной модели с клином.
54. Перекрытие: `split_overlap` в градусах для захвата полной толщины торца.
55. Клеевые блоки (`glue_blocks_raw`): только на внутренних линиях разреза.
56. Блоки содержат отверстия под шестигранные пины.
57. На крайних торцах (не склеиваемых) блоки ставятся без отверстий для усиления.
58. После склейки геометрия полностью соответствует цельному резаку.
59. Части выкладываются в ряд с шагом `size_x + part_gap`.
60. Для прямых цилиндров разрезка идет вертикальными полосами.
61. Разрезка дуг идет по углу клином.
62. Пины: диаметр `pin_d=5`, длина `pin_length=8`.
63. Отверстия под пины: `pin_d + pin_tolerance`.
64. Толеранс `pin_tolerance = 0.2` по умолчанию.
65. Пины печатаются отдельно, отверстия только в блоках.

### Раздел 5: Интерфейс и Customizer (пп. 66-80)
66. Все ключевые параметры выносятся в Customizer.
67. Параметры группируются по смыслу: `[ВЫБОР МОДЕЛИ]`, `[РЕЖИМ]`, `[ГЛИНА И УСАДКА]`, и т.д.
68. Для выбора модели использовать слайдер `mug_id` с диапазоном.
69. Избегать полей ввода с клавиатуры в Customizer.
70. Параметры, не предназначенные для пользователя, скрывать.
71. Использовать `[1:1:132]` для целочисленных слайдеров.
72. Для углов использовать шаг 0.1 или 1 градус.
73. Для булевых параметров использовать `[true, false]`.
74. Комментарии к параметрам в Customizer должны быть на русском.
75. Единицы измерения указывать в комментарии: `// [1:1:132]`.
76. Имена параметров — snake_case, без сокращений (кроме устоявшихся).
77. В путях `include` использовать относительные пути (`../library/mug_db.scad`).
78. Версия файла указывается в заголовке: `// @version X.Y_YYYY-MM-DD`.
79. `echo()` в начале рендера выводит ключевые параметры для проверки.
80. Все размеры в мм.

### Раздел 6: Контроль качества и отладка (пп. 81-100)
81. `assert()` для проверки всех критических размеров (H>0, D>0, Wall>0).
82. Проверка `effective_x > 0` и `effective_y > 0`.
83. Предпочтение отдается `diff()` вместо `difference()`.
84. Предпочтение отдается `offset()` вместо ручного построения контуров.
85. Проверка радиусов: `max(radius, 0.01)` всегда.
86. Приоритет: чистая консоль (нет warnings) > правильная геометрия.
87. При ошибке CGAL сначала `[АНАЛИЗ]`, потом `К9`.
88. После трех неудач предложить `К9_DEBUG`.
89. `К9_DEBUG` — упрощенный код без BOSL2, с базовыми примитивами.
90. Проверка размеров через `echo()` перед рендером.
91. Код должен работать в OpenSCAD Nightly.
92. Фаска на отверстиях — последним шагом, после проверки геометрии.
93. Все операции с `offset()` проверять на самопересечение при больших дельтах.
94. `$fn` для окружностей не менее 64, для дуг — 360.
95. Масштабирование (Zoom) применяется до компенсации усадки (Shrinkage).
96. Усадка: `final_scale = 1 + clay_shrink_pct / 100`.
97. Высота стенки: `final_height - clay_thickness`.
98. Толщина дна учитывается в общей высоте кружки.
99. Финальный рендер (F6) перед экспортом STL обязателен.
100. При любых сомнениях в геометрии использовать `[СОМНЕНИЕ]`.
